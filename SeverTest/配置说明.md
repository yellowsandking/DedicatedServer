# 类配置说明

## 重要：这些类不需要手动放到场景里！

### 自动管理的类（不需要手动放置）

以下类由Unreal Engine自动创建和管理，**不需要**手动拖拽到场景中：

1. **ServerTestGameMode** (游戏模式)
   - 在游戏启动时自动创建
   - 在代码中已设置为默认GameMode
   - 可以通过 `GetWorld()->GetAuthGameMode()` 访问

2. **ServerTestGameState** (游戏状态)
   - 由GameMode自动创建
   - 在代码中已配置（GameMode构造函数中设置）
   - 可以通过 `GetWorld()->GetGameState()` 访问

3. **ServerTestPlayerController** (玩家控制器)
   - 当玩家连接时自动创建（每个玩家一个）
   - 在代码中已配置（GameMode构造函数中设置）
   - 可以通过 `GetPlayerController()` 访问

### 需要手动放置的类

**ServerTestActor** (演示Actor)
- ✅ **需要手动放置到场景中**
- 这是一个演示用的Actor，展示服务器-客户端同步
- 在编辑器中：
  1. 打开你的地图
  2. 在Content Browser中找到 `ServerTestActor`（编译后会出现）
  3. 拖拽到场景中
  4. 或者使用 `Place Actors` 面板搜索并放置

## 如何验证配置是否正确

### 方法1：在编辑器中测试

1. 编译项目
2. 打开编辑器
3. 点击 `Play` -> `Number of Players` 设置为2
4. 选择 `Net Mode` 为 `Play As Listen Server`
5. 如果看到服务器日志输出，说明配置成功

### 方法2：检查代码

在 `ServerTestGameMode.cpp` 的构造函数中，已经设置了：
```cpp
PlayerControllerClass = AServerTestPlayerController::StaticClass();
GameStateClass = AServerTestGameState::StaticClass();
```

这意味着这些类会自动使用，无需额外配置。

### 方法3：在项目设置中检查（可选）

如果你想在项目设置中查看或修改：
1. 打开 `Edit` -> `Project Settings`
2. 在 `Game` -> `Default Modes` 中可以看到：
   - Default GameMode: `ServerTestGameMode`
   - Player Controller Class: `ServerTestPlayerController`
   - Game State Class: `ServerTestGameState`

## 总结

- ❌ **不需要**手动放置：GameMode, GameState, PlayerController
- ✅ **需要**手动放置：ServerTestActor（演示用）

所有配置已在代码中完成，编译后即可使用！
